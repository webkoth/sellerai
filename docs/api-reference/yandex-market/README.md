# Yandex Market Partner API

> API для продавцов Яндекс.Маркета — управление товарами, заказами и аналитикой

**Base URL:** `https://api.partner.market.yandex.ru`
**Generated:** 2024-12-28
**API Version:** v2
**Rate Limit:** 300 запросов / 60 секунд

---

## Обзор

Яндекс.Маркет Partner API позволяет автоматизировать управление магазином на маркетплейсе через программные интерфейсы.

### Модели размещения

Яндекс.Маркет поддерживает несколько моделей размещения:

| Модель | Описание | Хранение | Доставка | Документация |
|--------|----------|----------|----------|--------------|
| **FBS** | Fulfillment by Seller | Ваш склад | Маркет | [✅ fbs.md](./fbs.md) |
| **Business** | Общие методы | — | — | [✅ business.md](./business.md) |
| FBY | Fulfillment by Yandex | Склад Маркета | Маркет | ⏳ Планируется |
| Express | Экспресс-доставка | Ваш склад/точка | Маркет (срочно) | ⏳ Планируется |
| DBS | Delivery by Seller | Ваш склад | Вы сами | ⏳ Планируется |

**Общие методы для всех моделей**

- ⭐ Отзывы и вопросы — работа с обратной связью
- 📈 Буст продаж — продвижение товаров ставками
- 💬 Чаты — общение с покупателями
- 🏆 Индекс качества — метрики качества работы

**Всего методов:** ~70

### ✅ [FBS API](./fbs.md)
**Fulfillment by Seller — хранение у вас, доставка через Маркет**

Основные категории:
- 🏪 Магазины — настройки, информация
- 📦 Товары — ассортимент, скрытие/показ
- 📊 Остатки — обновление и просмотр остатков на складе
- 💰 Цены — управление ценами магазина
- 📋 Заказы — обработка, статусы, маркировка
- 🚚 Отгрузки — создание, подтверждение, документы
- 🏷️ Ярлыки — печать ярлыков на коробки
- ↩️ Возвраты — невыкупы и возвраты товаров
- 📈 Отчеты — детальная статистика
- ⭐ Качество — индекс качества магазина
- 🏢 Склады — управление складами

**Всего методов:** ~80

**Особенности FBS:**
- Вы храните товары на своем складе
- Обновляете остатки через API (минимум раз в час)
- Собираете и упаковываете заказы
- Передаете их Маркету по расписанию отгрузок
- Маркет доставляет покупателям

## Аутентификация

### API-Key Token (рекомендуется)

```http
GET /v2/campaigns
Api-Key: YOUR_API_KEY
Content-Type: application/json
```

**Создание токена:**
1. Войдите в [Личный кабинет продавца](https://partner.market.yandex.ru)
2. Нажмите на иконку аккаунта → **Настройки**
3. Раздел **API и модули** → **Токены авторизации**
4. Создайте токен с нужными правами доступа

**Права доступа:**
- `all-methods` — Полное управление кабинетом
- `marketplace` — Управление товарами и заказами
- `analytics` — Только отчеты
- `settings-management` — Настройки магазинов

### OAuth 2.0 (устаревший)

OAuth-токены поддерживаются, но **не рекомендуются**. Используйте API-Key.

## Идентификаторы

### businessId
**Идентификатор кабинета** — объединяет несколько магазинов.

Используется в:
- Business API (общие методы)
- Управление товарами в каталоге
- Установка цен для всех магазинов
- Отчеты по кабинету

### campaignId
**Идентификатор магазина** — конкретный магазин в рамках кабинета.

Используется в:
- Model-specific API (FBS, FBY, Express, DBS)
- Управление заказами
- Остатки на складе
- Отгрузки

**Получение идентификаторов:**
```http
GET /v2/campaigns
```

## Rate Limits

### Глобальные ограничения
- **Параллельные запросы:** не более 2 одновременных запросов с одного логина
- **Общий лимит:** 300 запросов / 60 секунд

### По типам методов

| Тип метода | Лимит |
|------------|-------|
| GET-запросы | 300 req/60s |
| POST-запросы (чтение) | 300 req/60s |
| PUT/POST (изменение) | 300 req/60s |
| Генерация отчетов | 10 req/60s |

**При превышении:**
- HTTP 429 Too Many Requests
- Заголовок `Retry-After` с временем ожидания

## Error Handling

### Формат ошибок

```json
{
  "status": "ERROR",
  "errors": [
    {
      "code": "INVALID_SKU",
      "message": "Товар с указанным SKU не найден"
    }
  ]
}
```

### Основные коды ошибок

| HTTP Code | Error Code | Описание | Решение |
|-----------|------------|----------|---------|
| 400 | INVALID_REQUEST | Некорректный запрос | Проверьте формат |
| 401 | UNAUTHORIZED | Неверный токен | Обновите токен |
| 403 | FORBIDDEN | Нет прав доступа | Проверьте права токена |
| 404 | NOT_FOUND | Ресурс не найден | Проверьте ID |
| 420 | METHOD_FAILURE | Ошибка выполнения | См. детали в errors |
| 429 | TOO_MANY_REQUESTS | Превышен лимит | Снизьте частоту запросов |
| 500 | INTERNAL_ERROR | Внутренняя ошибка | Повторите позже |

## Common Workflows

### 1. Добавление нового товара (FBS)

```
1. Business API: POST /v2/businesses/{businessId}/offer-mappings/update
   → Добавление товара в каталог кабинета

2. FBS API: POST /v2/campaigns/{campaignId}/offers/update
   → Добавление товара в конкретный магазин

3. FBS API: PUT /v2/campaigns/{campaignId}/offers/stocks
   → Установка остатков на складе

4. Business API: POST /v2/businesses/{businessId}/offer-prices/updates
   → Установка цены
```

### 2. Обработка заказа (FBS)

```
1. FBS API: GET /v2/campaigns/{campaignId}/orders
   → Получение новых заказов (статус PROCESSING)

2. FBS API: PUT /v2/campaigns/{campaignId}/orders/{orderId}/boxes
   → Указание упаковок

3. FBS API: PUT /v2/campaigns/{campaignId}/orders/{orderId}/status
   → Обновление статуса → READY_TO_SHIP

4. FBS API: POST /v2/campaigns/{campaignId}/first-mile/shipments/{shipmentId}/confirm
   → Подтверждение отгрузки

5. FBS API: GET /v2/campaigns/{campaignId}/orders/{orderId}/delivery/labels
   → Печать ярлыков
```

### 3. Обновление остатков (FBS)

```
PUT /v2/campaigns/{campaignId}/offers/stocks

Body:
{
  "skus": [
    {
      "sku": "SKU-123",
      "warehouseId": 12345,
      "items": [
        {
          "count": 50,
          "type": "FIT",
          "updatedAt": "2024-12-28T12:00:00Z"
        }
      ]
    }
  ]
}
```

**Важно:** Обновляйте остатки минимум раз в час!

## Best Practices

### 1. Регулярное обновление остатков (FBS)
✅ Обновляйте остатки **каждый час**
✅ При обнулении товар автоматически скрывается
✅ Используйте `warehouseId` для правильной привязки

### 2. Мониторинг заказов
✅ Проверяйте новые заказы **каждые 5-10 минут**
✅ Обновляйте статусы своевременно
✅ Подтверждайте отгрузки до дедлайна

### 3. Работа с ошибками
✅ Логируйте все ошибки API
✅ Используйте retry с exponential backoff
✅ Мониторьте rate limits через заголовки

### 4. Производительность
✅ Используйте batch-методы где возможно
✅ Кэшируйте справочники (категории, регионы)
✅ Делайте параллельные запросы (макс. 2)
