# Дизайн: единый template оверлеев «KOTELNIKOVARTIFACT»

**Дата:** 2026-06-18
**Область:** `remotionvideo/src/overlays/`
**Тип:** консолидация + один новый компонент (`Statement`) + сквозная витрина (`Template`).

## Цель

Свести curated-набор кадров/анимаций в единый переиспользуемый стилевой template:
кубики стиля + одна канонная сквозная витрина-эталон + чистая дока. Следующие
ролики собираются вручную из кубиков. `theme.ts` остаётся единой точкой стиля.

Решения, зафиксированные на брейншторме:
- **Форма:** Kit + витрина-эталон (не data-driven сборщик).
- **Роль KEY:** заменяется новым, более выразительным приёмом `Statement` (без кремовой карточки).
- **Темы:** `ember` + `cyan` равноправны; витрина регистрируется в обеих.
- **Витрина:** сквозная end-to-end **и** отдельные композиции-кубики для точечного рендера.
- `LogoReveal-Cosmic` — компонент №1 (canonical логотип); `Endcard` (статичный PNG) и `Key` (примитив) удаляются.

## Компонент `Statement`

Роль «короткая ключевая фраза», на космо-фоне без кремовой карточки. Берёт из
`OverlayItem`: `text` (с сегментом `*…*`), `kicker?`, `variant?`. Размер шрифта
адаптивен к длине строки (как в `Fact`/`Title`). Использует общие примитивы из
`kit.tsx` (`Centered`, `Appear`, `Kicker`, `HighlightWord`, `useSpringIn`).

**Вариант `center` (дефолт):**
- фраза по центру, sentence-case (отличие от КАПС-`Title`), вес 800;
- тонкая **акцентная черта со свечением** над фразой, «прочерчивается» `scaleX` spring;
- вход — пословный spring slide-up (stagger);
- сегмент `*…*` — маркерная подсветка-«прочерк» через `HighlightWord` + лёгкое свечение акцента.

**Вариант `quote` (пулл-квот):**
- left-aligned;
- **вертикальная акцентная черта слева**, растёт `scaleY` spring, со свечением (`glow`);
- фраза справа от черты, sentence-case, крупная, пословный slide-up;
- сегмент `*…*` — акцентный цвет + glow;
- опциональный `kicker` над фразой.

Выбор варианта — поле `variant` в `OverlayItem` (по аналогии с `SubtitleVariant`).
Остальные роли поле игнорируют.

## Композиция `Template` (сквозная витрина)

Одна композиция на **непрерывном фоне** (`Stage` — один на весь ролик, как уже
сделано в `ReelOverlays`), три акта:
1. **Интро** — `LogoRevealCore` поверх общего фона (без своей `Stage`), ~5–6 c;
2. **Роли** — по одной сцене `Title / Fact / Statement(center) / Statement(quote) / Type / Cta` через переходы (`fade`/`slide`/`wipe`);
3. **Субтитры** — демо-плашки `glass` + `solid` в финале.

Бренд-тег/шеврон во время интро скрыты (логотип сам несёт бренд), появляются на
акте ролей. Регистрируется в `Root.tsx` в обеих темах: `Template`, `Template-Cyan`.
Длительность — через `calculateMetadata` (сумма актов − перекрытия переходов).

**Секции-кубики остаются** отдельными композициями для точечного рендера:
`LogoReveal-Cosmic`, `Overlays-Showcase` (роли), `SubtitlePlates`.

## Темы

`ember` + `cyan` равноправны. `LogoRevealCosmic` принимает `theme` (дефолт `cyan`);
метеоры берут `glow` из акцента темы — на `ember` получится «огонь входа в атмосферу».
Дефолт зарегистрированной композиции `LogoReveal-Cosmic` — `cyan` (как сейчас).

  в Remotion Studio открываются `Template` / `Template-Cyan` и проигрывают все три акта;
  ни одна сцена в `reels.ts` не рендерится пустой.

## Вне области (YAGNI)

- Data-driven сборщик роликов из конфига — не делаем.
- Новые темы кроме ember/cyan — не делаем.
- Переписывание текстов роликов — не делаем (только переименование роли KEY→STATEMENT).
