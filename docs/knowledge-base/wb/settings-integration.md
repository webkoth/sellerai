## Профиль продавца и роли пользователей

### Раздел «Профиль» на портале продавца

**Что содержит:**
- Данные организации (ИНН, ОГРН, юр. адрес)
- Контактная информация
- Реквизиты для выплат
- Настройки профиля

**Разделы:**
- Общая информация
- Банковские реквизиты
- Контакты
- Документы организации

### Пользователи и доступы

**Управление пользователями:**
- Добавление сотрудников
- Назначение ролей
- Ограничение доступа

**Роли пользователей:**

**1. Администратор**
- Полный доступ ко всем функциям
- Управление пользователями
- Финансовые операции
- Изменение настроек

**2. Менеджер по продажам**
- Управление товарами
- Работа с заказами
- Ответы на отзывы
- Просмотр аналитики

**3. Аналитик**
- Доступ к отчётам
- Просмотр статистики
- Без возможности изменения товаров

**4. Контент-менеджер**
- Редактирование карточек
- Загрузка фото/видео
- Управление ценами
- Без доступа к финансам

**5. Логист**
- Управление поставками
- Работа с остатками
- Создание заказов на поставку
- Без доступа к ценам и финансам

**Настройка ролей:**
1. Перейти в «Пользователи и доступы»
2. Добавить нового пользователя
3. Указать email
4. Назначить роль
5. Пользователь получит приглашение

## Изменение данных продавца

### Как изменить данные продавца

**Что можно изменить:**
- Контактные данные
- Банковские реквизиты
- Адрес
- Контактное лицо

**Что нельзя изменить:**
- ИНН организации
- Организационно-правовую форму
- Для этого нужна перерегистрация

**Процедура изменения:**
1. Подать заявку на изменение
2. Приложить подтверждающие документы
3. Дождаться проверки (1-3 дня)
4. Получить подтверждение

### Как изменить номер телефона

**Процесс:**
1. Перейти в раздел «Профиль»
2. Нажать «Изменить телефон»
3. Ввести новый номер
4. Подтвердить через SMS-код
5. Новый номер вступит в силу

**Важно:**
- Телефон используется для входа
- Получения уведомлений
- Двухфакторной аутентификации

### Восстановление доступа к личному кабинету

**Если забыли пароль:**
1. Нажать «Забыли пароль?»
2. Ввести номер телефона/email
3. Получить код восстановления
4. Установить новый пароль

**Если потеряли доступ к телефону:**
1. Обратиться в поддержку WB
2. Предоставить подтверждающие документы
3. Пройти процедуру верификации
4. Получить новый доступ

## Безопасность данных

### Безопасность данных и профиля продавца

**Основные угрозы:**
- Взлом аккаунта
- Фишинг
- Утечка API-токенов
- Несанкционированный доступ

### Защита личного кабинета с помощью двухфакторной аутентификации

**Что это:**
- Дополнительный уровень защиты
- Вход по паролю + одноразовый код
- Защита от взлома

**Как включить:**
1. Перейти в «Безопасность»
2. Включить двухфакторную аутентификацию (2FA)
3. Выбрать способ получения кодов:
   - SMS на телефон
   - Приложение-аутентификатор (Google Authenticator, Authy)
4. Подтвердить настройку

**Рекомендуется:**
- Использовать приложение-аутентификатор (надёжнее SMS)
- Сохранить резервные коды

### Как изменить адрес электронной почты для двухфакторной аутентификации

**Процесс:**
1. Войти в раздел «Безопасность»
2. Перейти в настройки 2FA
3. Изменить email для уведомлений
4. Подтвердить через код на новый email

### Как безопасно работать с данными портала WB Partners

**Рекомендации:**

**1. Пароли:**
- Использовать сложные пароли (>12 символов)
- Не использовать один пароль для разных сервисов
- Менять пароль раз в 3-6 месяцев
- Использовать менеджер паролей

**2. Доступы:**
- Не передавать свой логин/пароль сотрудникам
- Создавать отдельные аккаунты для каждого сотрудника
- Ограничивать права доступа по ролям
- Удалять доступы при увольнении сотрудников

**3. API-токены:**
- Не публиковать токены в открытом доступе
- Использовать разные токены для разных задач
- Регулярно обновлять токены
- Хранить в безопасном месте

**4. Компьютер:**
- Использовать антивирус
- Не работать с чужих/публичных компьютеров
- Всегда выходить из аккаунта после работы
- Не переходить по подозрительным ссылкам

### Как защититься от мошенников

**Типичные схемы мошенничества:**

**1. Фишинг:**
- Поддельные письма от имени WB
- Просьба ввести логин/пароль на поддельном сайте
- Запрос токенов API

**Защита:**
- Проверять адрес отправителя
- Не переходить по ссылкам из писем
- Вводить данные только на официальном сайте (seller.wildberries.ru)

**2. Поддельная техподдержка:**
- Звонки от имени WB с просьбой назвать данные
- Запрос кода из SMS

**Защита:**
- WB никогда не просит пароли и коды по телефону
- При сомнениях - звоните на официальный номер поддержки

**3. Взлом аккаунта:**
- Изменение реквизитов для вывода средств
- Изменение цен на товары
- Отмена заказов

**Защита:**
- Включить двухфакторную аутентификацию
- Регулярно проверять настройки профиля
- Мониторить операции по балансу

## Интеграция и API

### Что такое WB API и как с ним работать

**WB API** - программный интерфейс для автоматизации работы с Wildberries.

**Возможности API:**
- Управление товарами (создание, редактирование)
- Получение остатков и цен
- Работа с заказами
- Получение отчётов и статистики
- Управление рекламными кампаниями

**Документация:**
- Официальная документация: https://openapi.wildberries.ru/
- Swagger спецификация
- Примеры запросов

### Категории данных WB API

**Основные категории:**

**1. Content (Контент)**
- Управление карточками товаров
- Загрузка фото и видео
- Редактирование характеристик

**2. Marketplace (Маркетплейс)**
- Получение остатков
- Цены и скидки
- Работа с поставками

**3. Statistics (Статистика)**
- Продажи
- Заказы
- Отчёты

**4. Analytics (Аналитика)**
- Воронка продаж
- Детальная аналитика
- Требует подписку «Джем»

**5. Promotion (Продвижение)**
- Управление рекламными кампаниями
- Статистика по рекламе
- Ставки и бюджеты

**6. Feedbacks (Отзывы)**
- Получение отзывов
- Ответы на отзывы

**7. Questions (Вопросы)**
- Получение вопросов
- Ответы на вопросы

### Как создать, обновить или удалить токен WB API

**Создание токена:**

1. Перейти в раздел «Интеграция» → «API»
2. Нажать «Создать токен»
3. Выбрать категории данных (права доступа):
   - Только чтение
   - Чтение и запись
4. Указать название токена (для идентификации)
5. Скопировать и сохранить токен

**⚠️ Важно:**
- Токен показывается только один раз при создании
- Сохраните его в безопасном месте
- Если потеряли - создайте новый

**Обновление токена:**
- Нельзя обновить существующий токен
- Нужно удалить старый и создать новый

**Удаление токена:**
1. Перейти в раздел «API»
2. Найти нужный токен
3. Нажать «Удалить»
4. Подтвердить удаление

**Рекомендации:**
- Создавать отдельные токены для разных задач
- Давать минимальные необходимые права
- Регулярно обновлять токены (раз в 3-6 месяцев)
- Удалять неиспользуемые токены

### Как подключиться по API через токен

**Использование токена:**

**1. В заголовке запроса:**
```
Authorization: {your_api_token}
```

**2. Пример запроса (curl):**
```bash
curl -X GET "https://suppliers-api.wildberries.ru/content/v2/get/cards/list" \
  -H "Authorization: your_api_token"
```

**3. Пример на Python:**
```python
import requests

headers = {
    'Authorization': 'your_api_token'
}

response = requests.get(
    'https://suppliers-api.wildberries.ru/content/v2/get/cards/list',
    headers=headers
)

print(response.json())
```

### Как подключиться по API через OAuth 2.0

**Что это:**
- Более безопасный способ аутентификации
- Не требует передачи токена напрямую
- Подходит для публичных приложений

**Процесс:**
1. Регистрация приложения в WB
2. Получение Client ID и Client Secret
3. Реализация OAuth 2.0 flow
4. Получение access token
5. Использование access token для запросов

**Документация:**
- Описание OAuth 2.0 flow в официальной документации API

## Частые ошибки

### Ошибка 1: Использование одного аккаунта для всех
- **Последствие:** Невозможность отследить действия сотрудников
- **Решение:** Создать отдельные аккаунты с ограниченными правами

### Ошибка 2: Отсутствие двухфакторной аутентификации
- **Последствие:** Высокий риск взлома
- **Решение:** Включить 2FA для всех аккаунтов

### Ошибка 3: Публикация API-токенов в коде
- **Последствие:** Утечка доступа, несанкционированные действия
- **Решение:** Хранить токены в переменных окружения, не коммитить в git

### Ошибка 4: Не отзывать доступы уволенных сотрудников
- **Последствие:** Бывшие сотрудники имеют доступ к аккаунту
- **Решение:** Немедленно удалять доступы при увольнении
