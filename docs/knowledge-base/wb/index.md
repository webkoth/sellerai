**Формат:** LLM-friendly markdown документы для быстрого доступа к информации.

## Как использовать базу знаний

### Для Skills (автоматически)

Skills автоматически обращаются к нужным разделам через Read tool при необходимости:

```python
# Пример: skill wb-seo-expert читает документацию по карточкам
Read("docs/knowledge-base/wb/product-cards.md")
```

**Преимущества:**
- Нет дублирования (единый источник истины)
- Всегда актуальная информация
- Skills остаются компактными (только логика, без embedded контента)

---

### Для ручного поиска

**1. Быстрая навигация:**
- Используйте разделы выше по категориям (начинающим, работа с товарами, продвижение и т.д.)

**2. Поиск по темам:**
- Ctrl+F (Cmd+F) по этому файлу для поиска ключевых слов
- Переход в нужный раздел markdown

**3. Курсы для обучения:**
- [courses/index.md](./courses/index.md) — полный каталог видеокурсов WB Pro

## API и техническая интеграция

**Важно:** API документация НЕ находится в knowledge-base, она остаётся в wb-mcp.

**Для работы с WB API используйте:**
- **wb-mcp** (MCP сервер) — реальные API вызовы через tools
- `mcp/wb-mcp/` — исходный код, документация API
- [Настройки и интеграция](./settings-integration.md) — как создать токены API

**Разделение ответственности:**
- `docs/knowledge-base/wb/` → справочная информация (как работать с WB)
- `mcp/wb-mcp/` → технические спецификации API, code, tools

## Обновления

**Частота обновления базы знаний:**
- Справочный центр WB: при изменениях в официальной документации (мониторинг ежемесячно)
- Курсы WB Pro: при появлении новых курсов (мониторинг ежемесячно)

**Как обновить:**
1. Использовать `marketplace-docs-parser` skill
2. Запустить Firecrawl scrape на обновлённые разделы
3. Обновить затронутые файлы в `docs/knowledge-base/wb/{category}.md`
4. Проверить актуальность `index.md`
5. Убедиться, что ссылки из skills работают

**Последнее обновление:** 2025-12-29
