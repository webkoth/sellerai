# Приёмка товара и учёт себестоимости (COGS)

Лёгкий управленческий учёт себестоимости, встроенный в синк. Без МойСклад/1С. Источник истины
по остаткам — наш сквозной пул; источник истины по себестоимости — `data/state/costs.json`.

## Модель

- Товар уникальный (1 шт), поэтому **одна себестоимость на изделие** (ключ — barcode WB).
- Себестоимость = материалы (камень/метеорит + бусины/шнур/фурнитура) + работа мастера — **одной суммой**.
- Прибыль = `цена − комиссия МП − себестоимость` (комиссия из категории, `data/mappings/pricing.json`).
- Налог НПД считается с выручки отдельно и в эту прибыль не входит.

## Команда приёмки (Telegram, в группе KOTELNIKOVARTIFACT)

```
/cost <штрихкод> <себестоимость> [заметка]
```
Примеры:
```
/cost 2052024009161 5000
/cost 2052024009161 5000 метеорит Алетай + работа
/c 2052024009161 5000           (короткий алиас)
```
Бот в ответ подтверждает и показывает **прогноз прибыли при выкупе** по WB/Ozon/ЯМ
(режим-A цена − комиссия − себестоимость).

> Латинская команда `/cost` ловится даже при privacy ON у бота. Натуральный ввод
> («приёмка 123 5000») потребует выключить privacy через BotFather — опционально.

## Процесс при поступлении нового товара

```
1. Мастер сделал изделие → на WB сгенерирован штрихкод
2. В группу:  /cost <штрихкод> <себестоимость>  → бот учёл + дал прогноз прибыли
3. Карточка на WB (мастер) с ценой = нужная маржа над себестоимостью
4. /sync → Ozon/ЯМ по режиму A (COGS уже привязан к barcode)
5. Продажа → алерт заказа показывает реальную ЧИСТУЮ прибыль (цена − комиссия − COGS)
6. 1-го числа → месячный P&L в группу (finance)
```

## Где что лежит

| Сущность | Файл / механизм |
|----------|-----------------|
| Каталог себестоимости | `data/state/costs.json` (ключ — barcode) |
| Аудит правок | `data/state/costs-log.jsonl` (append-only) |
| Offset Telegram | `data/state/tg-offset.json` |
| Приём команд | cron `* * * * * intake` (lock `sai-intake.lock`) |
| P&L | cron `0 10 1 * * finance` + по запросу `node ... finance [дней]` |
| Расчёт прибыли в алерте | `sync/src/commands/orders.ts` |
