# Кейс: Расчёт налоговой базы НПД для селлера маркетплейсов

## Задача

Клиент — самозанятый продавец ювелирных украшений из метеоритов на Wildberries и Яндекс.Маркете. Обратился с запросом:

1. Рассчитать точную налоговую базу НПД за 2024-2025 годы
2. Проверить, не превышен ли годовой лимит 2 400 000 ₽
3. Понять, какое поле из API маркетплейса использовать для расчёта налога
4. Получить помесячную разбивку для внесения чеков в "Мой налог"
5. Учесть возвраты — нужно ли их вычитать из налоговой базы

## Что сделали

### 1. Исследовали позицию ФНС по налоговой базе

Выяснили, что налоговая база НПД = сумма, которую заплатил покупатель (цена товара после скидки маркетплейса). Комиссия площадки НЕ уменьшает налоговую базу — это расходы продавца.

**Поле в API Wildberries:** `retail_amount` — цена для покупателя после СПП.

### 2. Получили данные из Wildberries API

Использовали endpoint `reportDetailByPeriod` с полной пагинацией. Обработали 3 254 записи за 2025 год.

**Результат по WB:**
- 2024 год (с 29.01): 101 073,72 ₽
- 2025 год: 2 272 182,90 ₽
- Продано единиц: 1 647 шт
- Возвратов: 120 шт (6,8%)

### 3. Разобрались с возвратами

Проанализировали код MCP-сервера и убедились, что возвраты уже исключены из налоговой базы на уровне фильтрации данных. Сумма `retail_amount` считается только по выкупленным позициям.

### 4. Получили данные с Яндекс.Маркета

API Яндекса ограничивает запросы интервалом 30 дней, поэтому запрашивали помесячно. Нашли 13 доставленных заказов во второй половине 2025 года.

**Результат по ЯМ 2025:** 82 203 ₽

## Выводы

1. Лимит НПД не превышен ни в одном году
2. Возвраты уже учтены в данных API — отдельно вычитать не нужно
3. Клиент может продолжать работать как самозанятый
4. До лимита в 2025 году осталось всего 45 614 ₽ — нужно мониторить
