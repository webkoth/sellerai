# Agent Skills — Мини-гайд по созданию

> Открытый стандарт от Anthropic для расширения возможностей AI-агентов

## Что такое Agent Skills?

**Agent Skills** — это папки с инструкциями, скриптами и ресурсами, которые AI-агенты могут автоматически обнаруживать и использовать. Формат принят Claude Code, Cursor, VS Code, GitHub Copilot и другими инструментами.

**Ключевая идея:** "Write once, use everywhere" — один skill работает во всех совместимых продуктах.

## Минимальная структура

```
skill-name/
└── SKILL.md          # Обязательный файл
```

С дополнительными ресурсами:

```
skill-name/
├── SKILL.md          # Инструкции для агента
├── scripts/          # Исполняемые скрипты
├── references/       # Дополнительная документация
└── assets/           # Шаблоны, данные, ресурсы
```

## Формат SKILL.md

### Обязательные поля frontmatter

```yaml
---
name: my-skill-name
description: Полное описание что делает skill и когда его использовать. Это поле критично для автоматической активации — агент решает загружать skill на основе description.
---
```

**Правила для `name`:**
- 1-64 символа
- Только lowercase, цифры и дефисы
- Не может начинаться/заканчиваться дефисом
- Без последовательных дефисов

**Правила для `description`:**
- 1-1024 символа
- Описывает ЧТО делает skill
- Описывает КОГДА использовать (триггеры)
- Чем точнее описание — тем лучше агент понимает когда активировать

### Опциональные поля

```yaml
---
name: pdf-processing
description: Extract text and tables from PDFs, fill forms, merge documents.
license: Apache-2.0
compatibility: "Requires Python 3.9+, poppler-utils"
metadata:
  author: example-org
  version: "1.0"
  category: document-processing
allowed-tools: Bash Read Write  # Экспериментальное поле
---
```

## Тело SKILL.md

После frontmatter идёт Markdown с инструкциями:

```markdown
---
name: my-skill
description: ...
---

# Название Skill

Краткое описание назначения.

## Ключевые понятия

Объяснение терминов и концепций.

## Инструкции

Пошаговые инструкции что делать.

## Примеры

Примеры входных данных и ожидаемого результата.

## Формат вывода

Как форматировать ответ пользователю.

## Ограничения

Что skill НЕ делает, edge cases.

## Источники

Откуда взяты знания.
```

## Принцип Progressive Disclosure

Agent Skills используют **прогрессивную загрузку** для экономии контекста:

```
1. Discovery     → Загружается только name + description (~50-100 токенов)
2. Activation    → При совпадении задачи загружается полный SKILL.md
3. Execution     → По необходимости загружаются scripts/ и references/
```

Это позволяет иметь много skills без перегрузки контекстного окна.

## Лучшие практики

### 1. Description — ключ к активации

```yaml
# Плохо — слишком общее
description: Помощь с Wildberries

# Хорошо — конкретные триггеры
description: Расчёт юнит-экономики товара на Wildberries. Используйте когда нужно рассчитать маржинальность, себестоимость, точку безубыточности, ROI рекламы. Помогает ответить на вопросы "сколько я зарабатываю", "выгоден ли товар", "какая маржа".
```

### 2. Структурированные инструкции

```markdown
## Алгоритм действий

1. Получить исходные данные
2. Проверить корректность
3. Выполнить расчёт
4. Отформатировать результат
5. Дать рекомендации
```

### 3. Примеры входа/выхода

```markdown
## Примеры

**Вход:** "Посчитай юнит-экономику для товара с ценой 1000₽ и себестоимостью 200₽"

**Выход:**
| Показатель | Значение |
|------------|----------|
| Цена продажи | 1000₽ |
| Себестоимость | 200₽ |
| Прибыль | 335₽ |
| Маржинальность | 33.5% |
```

### 4. Таблицы и формулы

```markdown
## Формулы

| Метрика | Формула | Целевое значение |
|---------|---------|------------------|
| Маржинальность | Прибыль / Выручка | > 20% |
| Оборачиваемость | Остаток / Продажи × дни | < 60 дней |
```

### 5. Скрипты для автоматизации

```
my-skill/
├── SKILL.md
└── scripts/
    └── calculate.py    # Агент может запустить через Bash
```

В SKILL.md:
```markdown
## Автоматизация

Для расчёта используй скрипт:
\`\`\`bash
python scripts/calculate.py --price 1000 --cost 200
\`\`\`
```

## Интеграция с Claude Code

### Расположение skills

```
project/
└── .claude/
    └── skills/
        ├── my-skill-1/
        │   └── SKILL.md
        └── my-skill-2/
            └── SKILL.md
```

### Как Claude Code находит skills

1. Сканирует `.claude/skills/` при старте
2. Загружает только `name` и `description` каждого skill
3. При запросе пользователя сопоставляет с descriptions
4. Загружает полный SKILL.md при совпадении
5. Выполняет инструкции из skill
