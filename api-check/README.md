# api-check

Самостоятельные проверки API маркетплейсов **Wildberries / Ozon / Яндекс.Маркет**.
Запускаешь сам — видишь по каждому эндпоинту: жив ли он (HTTP-код, время ответа),
сколько записей вернулось и пример данных. Только чтение, ничего не меняет.

Отдельное приложение на **TypeScript 7** (нативный компилятор `@typescript/native-preview` → `tsgo`).
Скрипты исполняются напрямую через Node (нативный type-stripping `.ts`, нужен **Node ≥ 24**, проект на **Node 26**).

## Установка

```bash
cd api-check
npm install
```

Токены берутся из `.env` в корне проекта (на уровень выше) — те же, что у MCP-серверов:
`WB_API_TOKEN`, `OZON_CLIENT_ID`, `OZON_API_TOKEN`, `YM_API_TOKEN`, `YM_BUSINESS_ID`, `YM_CAMPAIGN_ID`.

## Запуск

```bash
npm run wb      # все эндпоинты Wildberries
npm run ozon    # все эндпоинты Ozon
npm run ym      # все эндпоинты Яндекс.Маркет
npm run all     # всё подряд + сводка

npm run typecheck   # проверка типов через tsgo (TypeScript 7)
```

### Флаги

Передаются после `--`:

| Флаг | Действие |
|------|----------|
| `--full` | не обрезать примеры данных (показать ответ целиком) |
| `--json` | печатать сырой JSON каждого ответа |
| `--only <строка>` | запустить только пробы, где имя/категория содержит строку |
| `--quiet` / `-q` | только строки статусов, без примеров |

Примеры:

```bash
npm run wb -- --only prices        # только цены WB
npm run all -- --only stocks       # остатки по всем трём МП
npm run ozon -- --full             # Ozon с полными данными
npm run all -- --quiet             # компактная сводка
```

### Что показывает

```
✅ [prices     ] Цены и скидки              200    212 мс  n=10
     { "nmID": 123, "vendorCode": "ABC", ... }
❌ [analytics  ] Воронка (Джем)             403    180 мс
   ↳ нет подписки «Джем» (ожидаемо)
```

- ✅/❌ — успех/ошибка пробы
- `[категория]` — раздел API
- HTTP-код (зелёный 2xx / жёлтый 4xx / красный 5xx / `ERR` — сетевой сбой)
- время ответа
- `n=` — сколько записей вернулось
- ниже — пример данных (или тело ошибки)

**Код выхода:** `0` — все пробы прошли, `1` — были ошибки (удобно для cron/CI).

## Гео-блок РФ-API и прокси

РФ-API (`*.wildberries.ru`, `*.ozon.ru`, `*.market.yandex.ru`) бывают недоступны
напрямую (нет VPN / гео-блок) — тогда все пробы падают с `ERR · fetch failed`.

Решения:
1. **Включить VPN** с не-РФ → запускать обычно (`npm run all`).
2. **Через SOCKS5-relay** (VPS с не-РФ IP). Поднять туннель и задать прокси —
   приложение само направит весь трафик через него:

   ```bash
   # туннель (держать в отдельном окне)
   ssh -N -D 1080 root@<vps-ip>

   # запуск через прокси
   ALL_PROXY=socks5://127.0.0.1:1080 npm run all
   ```

   Поддерживаются `ALL_PROXY`, `HTTPS_PROXY`, `API_CHECK_PROXY`
   (схемы `socks5://`, `socks4://`, `http(s)://`; можно с `user:pass@`).
   Можно прописать `ALL_PROXY` прямо в `.env`.

   Пакеты `undici`/`socks` нужны только при использовании прокси.

## Структура

```
api-check/
├── package.json        # type:module, скрипты, tsgo (TS7)
├── tsconfig.json       # noEmit + allowImportingTsExtensions (запуск через node)
├── .nvmrc отсутствует  # версия Node берётся из окружения (проект на 26)
└── src/
    ├── lib.ts          # .env, http(), прокси, флаги, рендер, runSuite
    ├── check-wb.ts     # пробы Wildberries
    ├── check-ozon.ts   # пробы Ozon
    ├── check-ym.ts     # пробы Яндекс.Маркет
    └── check-all.ts    # все три + сводка
```

Эндпоинты и тела запросов взяты 1-в-1 из рабочих MCP-серверов (`mcp/*-mcp`),
поэтому совпадают с тем, что реально дёргают агенты.
