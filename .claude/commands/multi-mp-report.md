# Кросс-маркетплейсовый отчёт

Сводный отчёт по всем подключённым маркетплейсам: Wildberries, Ozon, Яндекс.Маркет.

## Алгоритм

### 1. Определение периода

```
week   → последние 7 дней
month  → последние 30 дней
custom → dateFrom...dateTo
```

### 2. Сбор данных

Параллельно запросить данные со всех подключённых маркетплейсов:

**Wildberries:**
```
wb_get_orders({ dateFrom, dateTo }) → заказы, выручка
wb_get_inventory({ mode: "all" }) → остатки
wb_get_reviews({ take: 20 }) → отзывы
wb_get_campaigns({ status: "active" }) → реклама
```

**Ozon:**
```
ozon_get_orders({ dateFrom, dateTo }) → заказы, выручка
ozon_get_stocks({}) → остатки
ozon_get_reviews({ limit: 20 }) → отзывы
```

**Яндекс.Маркет:**
```
ym_get_orders({ dateFrom, dateTo }) → заказы, выручка
ym_get_stocks({}) → остатки
ym_get_reviews({ limit: 20 }) → отзывы
ym_get_quality_rating({}) → качество
```

## Логика сравнения

### Распределение выручки

Рассчитай долю каждого маркетплейса:
```
доля_WB = выручка_WB / (выручка_WB + выручка_Ozon + выручка_YM) * 100%
```

### Приоритизация рекомендаций

| Тип | Приоритет | Пример |
|-----|-----------|--------|
| Нулевые остатки на активном МП | P1 | "Пополнить товар X на WB" |
| Негативные отзывы без ответа | P1 | "Ответить на отзыв на Ozon" |
| Низкий ROI рекламы | P2 | "Оптимизировать кампанию на WB" |
| Расхождение цен между МП | P2 | "Выровнять цены на товар X" |
| Отсутствие товара на одном из МП | P3 | "Добавить товар X на Яндекс.Маркет" |

## Обработка ошибок

Если один из маркетплейсов недоступен:
1. Отметить в отчёте: `⚠️ Ozon: данные недоступны`
2. Продолжить сбор данных с остальных МП
3. Пересчитать ИТОГО без недоступного МП
