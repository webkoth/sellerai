# Расчёт юнит-экономики

Используй skill `wb-unit-economics` для выполнения расчётов.

## Алгоритм

1. **Получение данных о товаре:**
   - Если указан артикул — получи данные через skill `wb-inventory`
   - Если не указан — запроси у пользователя данные или используй остатки

2. **Запроси недостающие данные у пользователя:**
   - Закупочная цена (себестоимость)
   - Цена продажи (или возьми из wb-inventory)
   - Категория товара (для определения комиссии)
   - Габариты/вес (для расчёта логистики)

3. **Рассчитай по формулам из skill:**
   - Комиссии WB (по категории)
   - Логистика (по габаритам)
   - Хранение (если применимо)
   - Эквайринг 2%
   - Налоги (УСН 6% или по режиму пользователя)

4. **Выведи результат в формате:**

```markdown
## Юнит-экономика: [Название товара]

### Входные данные
| Параметр | Значение |
|----------|----------|
| Артикул | ... |
| Закупочная цена | X₽ |
| Цена продажи | X₽ |
| Категория | ... |

### Расчёт расходов
| Статья расходов | Сумма | % от цены |
|-----------------|-------|-----------|
| Себестоимость | X₽ | X% |
| Комиссия WB | X₽ | X% |
| Логистика | X₽ | X% |
| Хранение | X₽ | X% |
| Эквайринг | X₽ | 2% |
| Налоги | X₽ | X% |
| **Итого расходов** | **X₽** | **X%** |

### Результат
| Показатель | Значение |
|------------|----------|
| Прибыль с единицы | X₽ |
| Маржинальность | X% |
| Рентабельность | X% |
| Точка безубыточности | X шт/мес |

### Оценка
[🟢 OK / 🟡 Низкая маржа / 🔴 Убыточно]

### Рекомендации
1. [Если есть проблемы — конкретные действия]
```

## Дополнительно

- Если маржинальность < 20% — предупреди о рисках
- Если убыточно — предложи варианты оптимизации (снижение себестоимости, повышение цены)
- Учитывай % выкупа для категории при расчёте реальной прибыли
