# Аналитика возвратов

Проанализируй возвраты за {{ period | default: "30d" }} на {{ marketplace | default: "all" }} маркетплейсах.

## Алгоритм

### 1. Получи данные о заказах и возвратах

{% if marketplace == "wb" or marketplace == "all" %}
**Wildberries:**
```
wb_get_orders dateFrom=[начало периода] dateTo=[сегодня]
Фильтр по статусам: canceled, canceled_by_client, defect
```
{% endif %}

{% if marketplace == "ozon" or marketplace == "all" %}
**Ozon:**
```
ozon_get_orders dateFrom=[начало периода] dateTo=[сегодня] status=cancelled
```
{% endif %}

{% if marketplace == "ym" or marketplace == "all" %}
**Яндекс.Маркет:**
```
ym_get_orders dateFrom=[начало периода] dateTo=[сегодня] status=CANCELLED
```
{% endif %}

### 2. Классифицируй возвраты по причинам

| Категория | Причины | Влияние |
|-----------|---------|---------|
| **Качество** | Брак, дефект, не соответствует описанию | Проблема с поставщиком/производством |
| **Размер/Цвет** | Не подошёл размер, цвет отличается | Проблема с карточкой/фото |
| **Логистика** | Повреждён при доставке, долгая доставка | Проблема со складом/перевозчиком |
| **Цена** | Нашёл дешевле, передумал | Конкурентная среда |
| **Прочее** | Не указана, отмена клиентом | Импульсные покупки |

### 3. Рассчитай метрики

**Общие метрики:**
```
Всего_заказов = количество заказов за период
Всего_возвратов = количество отменённых/возвращённых
Процент_возвратов = Всего_возвратов / Всего_заказов × 100%
```

**Финансовые потери:**
```
Сумма_возвратов = сумма отменённых заказов
Упущенная_выручка = Сумма_возвратов
Затраты_на_логистику = (возвраты с доставкой) × тариф_логистики
Итого_потери = Упущенная_выручка + Затраты_на_логистику
```

**По товарам:**
```
Для каждого товара:
  Заказов = количество заказов
  Возвратов = количество возвратов
  Процент = Возвратов / Заказов × 100%
  Сумма_возвратов = стоимость возвращённых товаров
```

### 4. Выяви проблемные товары

Критерии проблемных товаров:
- Процент возвратов > 15% (при заказах > 10)
- Сумма возвратов > 50 000 ₽
- Повторяющиеся причины (3+ одинаковых)

### 5. Сформируй рекомендации

| Проблема | Рекомендация |
|----------|--------------|
| Высокий % возвратов по размеру | Добавить размерную сетку, фото на модели |
| Жалобы на качество | Проверить партию, сменить поставщика |
| Цвет не соответствует | Обновить фото при естественном освещении |
| Долгая доставка | Распределить остатки по региональным складам |
| Много отмен до получения | Ускорить сборку, уведомлять о статусе |

## Интеграция с другими командами

После анализа возвратов рекомендуется:

1. **/seo-audit** — для товаров с причиной "не соответствует описанию"
2. **/funnel** — для понимания, на каком этапе теряются покупатели
3. **/competitors** — сравнить свои возвраты с конкурентами
4. **/review-reply** — ответить на негативные отзывы о возвратах
