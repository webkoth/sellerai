# Генерация ответа на отзыв/вопрос

Используй skill `wb-reviews-responder` для генерации ответа.

## Алгоритм

1. **Анализ входных данных:**
   - Определи тип: отзыв или вопрос
   - Если отзыв — определи тональность (позитивный/негативный/нейтральный)
   - Если указана оценка — используй её для контекста

2. **Классификация отзыва (если применимо):**

   | Тип негатива | Признаки | Стратегия ответа |
   |--------------|----------|------------------|
   | Не по теме (ПВЗ, доставка) | Жалоба на WB | Вежливо перенаправить в поддержку WB |
   | Обоснованная претензия | Реальная проблема с товаром | Извиниться, предложить решение |
   | Брак/дефект | Описание дефекта | Предложить связаться для замены |
   | Несоответствие ожиданиям | "Думал будет другое" | Уточнить описание, выразить сожаление |
   | Заказной негатив | Неадекватный текст, без конкретики | Рекомендовать жалобу в WB |

3. **Генерация ответа:**

   **Для позитивного отзыва:**
   - Поблагодарить
   - Выразить радость
   - Пригласить снова

   **Для негативного отзыва:**
   - Поблагодарить за обратную связь
   - Выразить сожаление (НЕ оправдываться)
   - Предложить решение или контакт
   - Показать заботу о клиенте

   **Для вопроса:**
   - Поприветствовать
   - Ответить по существу
   - Предложить дополнительную помощь

4. **Выведи результат в формате:**

```markdown
## Ответ на отзыв/вопрос

### Анализ

| Параметр | Значение |
|----------|----------|
| Тип | Отзыв / Вопрос |
| Тональность | Позитивный / Негативный / Нейтральный |
| Оценка | ⭐ X (если есть) |
| Категория проблемы | [если негативный] |

### Исходный текст
> [текст отзыва/вопроса]

### Рекомендуемый ответ

```
[текст ответа]
```

### Дополнительные действия
- [ ] [Если нужно что-то исправить в карточке]
- [ ] [Если нужно связаться с клиентом]
- [ ] [Если нужно пожаловаться на отзыв]
```

## Правила генерации ответов

1. **Длина:** 2-4 предложения (короткие, ёмкие)
2. **Тон:** Вежливый, профессиональный, без заискивания
3. **Без:** Капса, множественных восклицательных знаков, эмодзи
4. **Персонализация:** По возможности обратиться к конкретике отзыва

## Шаблоны-основы

**Позитивный:**
```
Благодарим за отзыв! Рады, что [конкретика из отзыва].
Будем рады видеть вас снова!
```

**Негативный (качество):**
```
Благодарим за обратную связь. Приносим извинения за доставленные неудобства.
Мы обязательно учтём ваши замечания. Для решения вопроса свяжитесь с нами: [контакт]
```

**Негативный (не по теме):**
```
Спасибо за отзыв. К сожалению, [описанная проблема] не относится к нашему товару —
это зона ответственности Wildberries. Рекомендуем обратиться в поддержку WB.
```

**Вопрос:**
```
Здравствуйте! [Ответ на вопрос].
Если остались вопросы — с удовольствием поможем!
```

## После генерации

- Если негативный отзыв указывает на реальную проблему — предложи обновить карточку
- Если вопрос типичный — предложи добавить информацию в описание/инфографику
- Если заказной негатив — предложи подать жалобу
