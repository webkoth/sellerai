# Прогнозирование спроса

Skill для прогнозирования продаж на основе исторических данных заказов.

## Когда использовать

- Планирование закупок
- Оптимизация складских остатков
- Предсказание сезонного спроса
- Расчёт точки перезаказа
- Оценка потенциала нового товара

## Источники данных

### Wildberries
```
wb_get_orders — история заказов за период
wb_get_inventory — текущие остатки
wb_get_sales_funnel — воронка (просмотры → заказы)
```

### Ozon
```
ozon_get_orders — история заказов
ozon_get_stocks — остатки FBO/FBS
```

### Яндекс.Маркет
```
ym_get_orders — история заказов
ym_get_stocks — остатки по складам
```

## Алгоритмы прогнозирования

### 1. Скользящее среднее (Moving Average)

Простой метод для товаров со стабильным спросом.

```
Прогноз = (Продажи за N дней) / N

Где N = 7, 14, 30 дней в зависимости от волатильности
```

**Когда использовать:**
- Товары без выраженной сезонности
- Стабильный спрос ±20%
- Базовые товары повседневного спроса

### 2. Взвешенное скользящее среднее (Weighted MA)

Больший вес недавним данным.

```
Прогноз = (W1 × Неделя1 + W2 × Неделя2 + W3 × Неделя3 + W4 × Неделя4) / (W1+W2+W3+W4)

Рекомендуемые веса: W1=4, W2=3, W3=2, W4=1
```

**Когда использовать:**
- Товары с трендом (рост/падение)
- После изменения цены или SEO
- Новые товары (3+ недель продаж)

### 3. Сезонный коэффициент

Учёт сезонных колебаний.

```
Сезонный_коэф = Продажи_месяца / Среднемесячные_продажи

Прогноз = Базовый_прогноз × Сезонный_коэф_следующего_месяца
```

**Сезонность по категориям:**

| Категория | Пик | Спад |
|-----------|-----|------|
| Одежда/обувь | Сентябрь, Март | Январь, Июль |
| Электроника | Ноябрь-Декабрь | Февраль |
| Товары для дома | Март-Май | Июль |
| Спорт/туризм | Май-Июнь | Ноябрь-Январь |
| Подарки | Декабрь, Февраль, Март | Апрель-Май |
| Школьные товары | Август-Сентябрь | Ноябрь-Июль |

### 4. Экспоненциальное сглаживание (Holt-Winters)

Для товаров с трендом и сезонностью.

```
Прогноз = (α × Факт) + (1-α) × Прогноз_пред

α (альфа) = 0.2-0.3 для стабильных товаров
α = 0.4-0.5 для волатильных
```

## Расчёт точки перезаказа

```
Точка_перезаказа = (Средние_продажи_в_день × Срок_поставки) + Страховой_запас

Страховой_запас = Средние_продажи_в_день × Коэф_безопасности × √Срок_поставки

Коэф_безопасности:
- 1.28 = 90% уровень сервиса
- 1.65 = 95% уровень сервиса
- 2.33 = 99% уровень сервиса
```

## Факторы, влияющие на прогноз

### Повышающие коэффициенты

| Фактор | Коэффициент |
|--------|-------------|
| Запуск рекламы | ×1.3-2.0 |
| Участие в акции | ×1.5-3.0 |
| Снижение цены 10%+ | ×1.2-1.5 |
| SEO-оптимизация | ×1.1-1.3 |
| Праздничный период | ×1.5-5.0 |
| Новые фото/видео | ×1.1-1.2 |

### Понижающие коэффициенты

| Фактор | Коэффициент |
|--------|-------------|
| Повышение цены | ×0.7-0.9 |
| Окончание акции | ×0.5-0.8 |
| Конец сезона | ×0.3-0.6 |
| Новый конкурент | ×0.8-0.95 |
| Out of stock > 3 дней | ×0.9 (потеря позиций) |

## Оценка точности прогноза

### MAPE (Mean Absolute Percentage Error)

```
MAPE = (1/n) × Σ|Факт - Прогноз| / Факт × 100%

Интерпретация:
< 10% — Отличный прогноз
10-20% — Хороший прогноз
20-30% — Приемлемый прогноз
> 30% — Требует корректировки
```

### Причины низкой точности

1. **Недостаточно данных** — менее 30 дней продаж
2. **Высокая волатильность** — товар нестабильный
3. **Внешние факторы** — акции, конкуренты, сезон
4. **Out of stock** — история искажена отсутствием товара

## Пример использования

### Запрос
```
Спрогнозируй спрос на товар nmId 123456789 на следующий месяц
```

### Алгоритм действий

1. **Получить историю заказов**
   ```
   wb_get_orders dateFrom=YYYY-MM-DD (90 дней назад)
   ```

2. **Получить текущие остатки**
   ```
   wb_get_inventory
   ```

3. **Рассчитать базовый прогноз**
   - Агрегировать продажи по неделям
   - Выбрать метод (MA/WMA/Сезонный)
   - Применить коэффициенты

4. **Скорректировать на факторы**
   - Проверить участие в акциях
   - Учесть сезонность
   - Проверить изменения цены

5. **Рассчитать рекомендации**
   - Дней до обнуления
   - Точка перезаказа
   - Объём заказа

6. **Оценить достоверность**
   - Проверить историю на out of stock
   - Оценить волатильность
   - Указать уровень уверенности

## Ограничения

- **Минимум данных:** 14 дней продаж для базового прогноза, 90 дней для сезонного
- **Не учитывает:** внезапные тренды, вирусный спрос, действия конкурентов
- **Out of stock:** периоды отсутствия товара искажают историю
- **Новые товары:** прогноз на основе аналогов из категории
