# Database Migrations

Миграции для базы данных SellerAI PostgreSQL.

## Порядок применения

Миграции должны применяться строго в указанном порядке:

1. **001_init.sql** — Инициализация БД
   - Создание основных таблиц (products, reviews, campaigns)
   - Настройка индексов
   - Базовые функции

2. **002_add_orders.sql** — Добавление заказов
   - Таблица orders
   - Связи с products
   - Индексы для производительности

3. **003_seller_settings.sql** — Настройки продавца
   - Таблица seller_settings
   - Конфигурация маркетплейсов
   - Пользовательские настройки

## Применение миграций

```bash
# Применить все миграции по порядку
psql -d sellerai -f migrations/001_init.sql
psql -d sellerai -f migrations/002_add_orders.sql
psql -d sellerai -f migrations/003_seller_settings.sql
```

## Откат

Каждая миграция должна иметь возможность отката. Создавайте файлы `XXX_rollback.sql` для критичных изменений.

## Новые миграции

При создании новой миграции:
1. Используйте следующий номер (004, 005, и т.д.)
2. Добавьте описание в этот README
3. Убедитесь что миграция идемпотентна (можно запускать повторно)
4. Протестируйте на тестовой БД перед применением
